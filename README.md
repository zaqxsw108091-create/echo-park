# 에코 파크 (Echo Park)

> 1~4인 협동 퍼즐 플랫포머 — 과거의 내 행동을 루프로 재생하는 **에코 시스템**으로 혼자서도, 친구와 함께도 풀 수 있는 협력 퍼즐 게임. 설치 없이 브라우저에서 바로 실행됩니다.

## 핵심 구현

### EchoRec 시스템 (핵심 퍼즐 메커닉)
플레이어의 행동을 최대 3개 슬롯에 녹화해 루프로 재생하는 시스템.

- `[E]` 키로 녹화 시작 → 다시 `[E]`로 정지 & 재생 시작
- 에코가 버튼 위에 위치하면 지속적으로 눌린 상태 유지
- 슬롯 3개(E1 / E2 / E3) 순서로 할당, `[R]` 키로 전체 리셋
- 에코만으로 버튼 3개까지 동시 커버 가능 → 솔로 클리어의 핵심

### NPC AI (5개 캐릭터)
각 NPC는 고유한 행동 특성을 가진 상태 머신(`PATROL → GOTO_BTN → PRESS → REACT`)으로 동작.

| 캐릭터 | 타입 | 이동속도 | 오차율 | 특성 |
|---|---|---|---|---|
| 🟣 루나 | 가이드형 | 130 | 4% | 막히면 힌트 제공, 기다림 |
| 🟢 리코 | 돌격형 | 200 | 18% | 빠르고 충동적, 선돌격 |
| 🟡 모카 | 분석형 | 105 | 2% | 정확한 타이밍 신호 |
| 🔴 블레이즈 | 스피드형 | 260 | 12% | 최고속도, 항상 전진 |
| 🩷 피치 | 서포터형 | 150 | 5% | 응원 특화, 부스트 지원 |

### 협동 퍼즐 설계
- **버튼 동시 눌림** 원칙: 출구는 지정된 버튼 전부 눌려야만 개방
- **NPC 부스트**: 특정 버튼은 점프로 도달 불가 → 동료 옆 `[F]`키로 발판 부스트
- `[T]` 키로 NPC에게 명령(이동 / 기다려 / 나한테 와) 가능
- 스테이지별로 최적 NPC 수 설계 (솔로 ~ 4인 모두 클리어 가능)

### 절차적 오디오
Web Audio API로 외부 파일 없이 모든 사운드 생성.

- **BGM**: 4-chord 진행 + 멜로디를 Oscillator로 실시간 생성, 4.5초 루프
- **빗소리**: Pink noise 알고리즘(Voss-McCartney)으로 생성한 앰비언트
- **SFX**: 점프 / 버튼 / 에코 / 클리어 / 저장 등 16종 개별 구현

### 세이브 시스템
- `localStorage` 기반 자동 저장 + 수동 슬롯 3개
- 슬롯에 스테이지·클리어 횟수·저장 시각 표시
- 불러오기 시 버튼 상태·에코 슬롯·NPC 위치까지 복원

### 우정 포인트 & 레벨업
- NPC별 우정 포인트 적립 (0 / 300 / 1000 / 3000 / 6000 임계값)
- 레벨업 시 해금 이벤트 (`SND.unlock()` + 파티클 연출)
- 메인 메뉴에서 각 NPC의 현재 레벨 실시간 표시

## 스테이지 구성

| 월드 | 테마 | 스테이지 수 | 최대 버튼 수 |
|---|---|---|---|
| World 1 | 에코 숲 | 4 (1-1 ~ 1-4) | 5개 |
| World 2 | 기어 공장 | 3 (2-1 ~ 2-3) | 6개 |
| World 3 | 에코 우주 | 준비 중 | — |

- 스테이지 1-4 「NPC 사다리」: 높은 발판 위 버튼은 NPC 부스트 없이 도달 불가
- 스테이지 2-3 「기어 마스터」: 버튼 6개 — 에코 3개 + NPC 3명 전원 필요

## 조작법

| 입력 | 동작 |
|---|---|
| `← →` / `A D` | 이동 |
| `↑` / `W` / `Space` | 점프 |
| `E` | 에코 녹화 시작 / 정지 |
| `R` | 전체 리셋 (에코·버튼·위치) |
| `T` | NPC에게 말 걸기 |
| `F` | NPC 옆에서 부스트 요청 |
| `ESC` / `P` | 일시 정지 |
| 📱 모바일 | 화면 왼쪽 터치: 이동 / 오른쪽: 점프 |

## 기술적 주의사항

- 렌더러를 `Phaser.CANVAS`로 고정한 이유: 일부 모바일·저사양 환경에서 WebGL 컨텍스트 손실(`CONTEXT_LOST_WEBGL`) 발생 방지
- 빗소리 생성을 `setTimeout 800ms` 지연 처리: 게임 초기화 중 오디오 버퍼 생성이 메인 스레드를 블로킹하는 문제 방지
- 파티클은 오브젝트 풀(pool) 방식으로 관리하여 GC 압력 최소화
- 출구 타일은 항상 `row 16` (바닥 레벨) 고정: 버튼 해제 후 걸어서 바로 진입 가능하도록 설계
